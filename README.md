# ChatGPT Web Gen

A single-file CLI tool that generates images via **ChatGPT Web** using CloakBrowser (stealth Playwright Chromium). No API key needed — just a ChatGPT Plus/Pro account.

```bash
pip install -r requirements.txt
python gen.py --login
python gen.py "a cinematic cyberpunk city at night"
python gen.py "put this product on a tropical beach" --ref product.png
```

---

## How it works

```
gen.py
  │
  ├── 1. Launch headless Chromium (CloakBrowser)
  ├── 2. Restore saved ChatGPT session cookies
  ├── 3. Open fresh conversation
  ├── 4. Upload reference image (optional)
  ├── 5. Type prompt + send
  ├── 6. Wait for generation (poll streaming indicator)
  ├── 7. Download the generated image
  └── 8. Print IMAGE:path or ERROR:reason
```

All in one command — no servers, no daemons, no API setup.

## Setup

### Requirements

- **Python 3.10+**
- **ChatGPT Plus, Pro, or Team account** — image generation requires a paid plan
- ~500MB disk for Chromium browser engine (one-time download)

### Install

```bash
git clone https://github.com/lunkerchen/chatgpt-web-gen.git
cd chatgpt-web-gen

# Create virtual environment (recommended)
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Download Chromium browser engine
playwright install chromium
```

### One-time login

```bash
python gen.py --login
```

This opens a **visible** Chrome window to `chat.openai.com`.  
Log in with your account, then **press Enter** in the terminal.

The session cookies are saved to `cookies.json` for subsequent headless use.

> **Cookie lifetime:** ~1-2 months (OpenAI policy).  
> When expired, re-run `python gen.py --login`.  
> The old `cookies.json` is overwritten automatically.

## Usage

### Text-to-image

```bash
python gen.py "a cute orange cat sitting on a desk, digital art style"
python gen.py "大稻埕碼頭黃昏，夕陽染紅天空，寫實攝影"
python gen.py "cinematic shot of a cyberpunk city, neon rain, 8k"
```

### Image-to-image (with reference)

```bash
python gen.py "redesign this logo in minimalist style" --ref logo.png
python gen.py "put this product on a tropical beach" --ref product.jpg
python gen.py "turn this sketch into a realistic painting" --ref sketch.png
```

### Output format

| Status | Output |
|--------|--------|
| Success | `IMAGE:/absolute/path/to/gen_1234567890.png` |
| Failure | `ERROR:<description>` (exits with code 1) |

Generated images land in `./temp/`.

## Troubleshooting

### `ModuleNotFoundError: No module named 'cloakbrowser'`

Dependencies not installed:

```bash
pip install -r requirements.txt
playwright install chromium
```

### `ERROR:No session`

No `cookies.json` found. Run:

```bash
python gen.py --login
```

### `ERROR:Session expired`

Cookies expired. Re-login:

```bash
python gen.py --login
```

### Browser launches but login page doesn't load

Check your internet connection. The tool needs to reach `chat.openai.com`.

### Image generation hangs or times out

Possible causes:

- ChatGPT is generating a slow response (can take 60-90s)
- The UI has changed — selectors may be outdated (see below)
- ChatGPT's servers are under load (rare)

The script waits up to 150 seconds before giving up.

### Generated image is low quality

ChatGPT Web controls the output resolution based on your account tier and the model it uses. This tool just downloads whatever ChatGPT produces.

## Selector maintenance

ChatGPT's frontend changes periodically. When generation breaks, the `S` dictionary at the top of `gen.py` likely needs updating:

| Key | Purpose | Typical DOM selector |
|-----|---------|---------------------|
| `chat_input` | Prompt input field | `#prompt-textarea`, `div[contenteditable="true"]` |
| `send` | Send button | `button[data-testid="send-button"]` |
| `generated` | Generated image detection | `img[alt*="已產生"]` |
| `streaming` | Generation-in-progress indicator | `button[data-testid="stop-button"]` |
| `file_input` | File upload input | `input[type="file"]` |
| `assistant` | Assistant message (failure text) | `[data-message-author-role="assistant"]` |
| `logged_in` | Login state check | `[data-testid="earth-icon"]`, `#prompt-textarea` |

To debug selector issues:

```bash
# Uncomment gen.py's debug line to save screenshots
# Or run headless=false temporarily
```

## Architecture notes

```
                   ┌──────────────────────┐
                   │    gen.py (CLI)      │
                   │                      │
  prompt ─────────>│  Playwright browser  │──> chat.openai.com
  --ref ──────────>│  (headless Chromium) │
                   │                      │<── generated image
                   └──────────────────────┘
                              │
                              ├── cookies.json ── session persistence
                              └── temp/ ──────── output images
```

The tool spawns a **fresh browser instance on every run**. There's no persistent daemon or background process. Each invocation is self-contained:

1. Launch Chromium (~8s cold start)
2. Restore cookies
3. Generate image (~30-60s)
4. Save cookies (for next time)
5. Close browser
6. Print result path

This simplicity comes at the cost of startup overhead but eliminates all state management issues.

## Limitations

- **Requires a paid ChatGPT account** (Plus/Pro/Team)
- **Single-file/CLI only** — no server, no API, no queue
- **Single-threaded** — one generation at a time (ChatGPT Web limitation)
- **Cookie-dependent** — session expires every 1-2 months
- **UI-fragile** — ChatGPT frontend changes may break selectors
- **Cold start** — each run launches a new browser (~8s overhead)
